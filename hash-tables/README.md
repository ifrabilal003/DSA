# Hash Tables

## Problem List
| # | Problem | Difficulty |
|---|---------|------------|
| 1 | Contains Duplicate | Easy |
| 2 | Two Sum | Easy |
| 3 | Valid Anagram | Easy |

## Approach
In brute force, I used nested loops which was slow. In optimal, I used HashMap and HashSet for fast lookup.

## Complexity Table
| Problem | Brute | Optimal |
|---------|-------|---------|
| Contains Duplicate | O(n^2) | O(n) |
| Two Sum | O(n^2) | O(n) |
| Valid Anagram | O(n log n) | O(n) |

## How to Run
javac TwoSum.java && java TwoSum
